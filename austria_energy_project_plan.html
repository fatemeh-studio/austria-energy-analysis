Project 3 plan: Austrian energy & climate analysis — datasets, research questions, folder structure, and build phases

Datasets

Primary · hourlyENTSO-E Transparency Platform
Hourly electricity generation by fuel type (hydro, wind, solar, gas, …), load, and day-ahead prices for Austria (AT). Free API after email registration. Python client: entsoe-py.
2015 → present15-min / hourlyREST APIentsoe-py
Primary · annualOur World in Data — Energy dataset
Long-run annual time series: energy mix, renewables share, CO₂ intensity, per-capita consumption. Single CSV from GitHub, Austria filter. Great for trend and context analysis.
1965 → 2024annualowid/energy-data on GitHub
Secondary · climateCopernicus / ERA5 (via Open-Meteo)
Daily/hourly weather: temperature, solar radiation, wind speed. Open-Meteo provides a free, no-key REST wrapper around ERA5 reanalysis data — no account needed.
1940 → presenthourlyopen-meteo.comno API key
Secondary · emissionsEurostat — GHG emissions (env_air_gge)
National GHG emissions by sector (energy, transport, industry). Annual, machine-readable JSON/CSV via Eurostat REST API.
1990 → 2022annualEurostat API

Research questions

RQ1
How has Austria's electricity generation mix shifted from fossil to renewable sources since 2015?
trend analysis · stacked area chart · annual aggregation
RQ2
Does temperature explain electricity demand patterns — and does that relationship change seasonally?
correlation · OLS regression · seasonal decomposition (STL)
RQ3
What is the "duck curve" signature of solar in Austria, and how pronounced has it become as capacity grew?
diurnal profiles · hourly aggregation · year-over-year comparison
RQ4
Are day-ahead electricity prices negatively correlated with renewable generation share? (Merit-order effect)
regression · scatter + density · partial correlation
RQ5
Is Austria's GHG intensity of electricity declining at a pace consistent with the 100% renewables by 2030 target?
goal tracking · log-linear trend · confidence intervals

Build phases

Phase 1
Setup & data collection
Repo init, ENTSO-E API key, fetch raw CSVs, Open-Meteo pull, OWID download. Notebook 01.
Phase 2
SQL schema + cleaning
Load data into DuckDB (your SQL on-ramp). Write CREATE TABLE + INSERT from CSV, type-cast, handle nulls. Notebook 02.
Phase 3
EDA
Distributions, missingness, seasonal patterns. SQL aggregations + pandas. Shared viz.py helpers begin here.
Phase 4
RQ analysis (×5)
One notebook per question. Mix of SQL queries, statsmodels, matplotlib. Each ends with a clear visual + 2-sentence finding.
Phase 5
Refactor to src/
Extract repeated logic into data_loader.py, clean.py, viz.py. First real OOP practice: DataLoader class with methods per source.
Phase 6
README + polish
Write a strong README with context, key findings, how to reproduce. Add requirements.txt. GitHub push.
